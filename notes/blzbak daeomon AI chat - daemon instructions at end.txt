The rsync destination is determined through a configuration chain:

## 1. Global Server Config (`.config` file)

Located next to the blzbak executable, contains:
```yaml
server:
  host: 192.168.1.100
  port: 7890
  ssh_user: backup
  backup_base: /blzbak      # Base path on server
```

## 2. Backup Set Creation

When you run `blzbak set create mybackup /path/to/source`, it:
- Reads the global server config
- Builds `dest_path = backup_base + "/" + set_name`
  - Example: `/blzbak/mybackup`
- Saves this in `sets/mybackup.yaml`:

```yaml
name: mybackup
server:
  host: 192.168.1.100
  port: 7890
  ssh_user: backup
  dest_path: /blzbak/mybackup   # Saved per set
source_paths:
  - /path/to/source
schedule: "0 2 * * *"
```

## 3. Cron Job Execution

When the cron runs `blzbak backup run mybackup`:
1. Loads `sets/mybackup.yaml`
2. Calls `server.rsync_dest("C")` which builds:
   ```
   ssh_user@host:dest_path/C/
   → backup@192.168.1.100:/blzbak/mybackup/C/
   ```
3. Runs rsync with that destination

See models.py where `rsync_dest()` builds the full rsync destination string, and set_cmd.py where `dest_path` is constructed during set creation.

## Usage

Now backup sets will be stored as `/etc/blzbak/<setname>.yaml`

**Important:** Since `/etc/blzbak` requires elevated permissions, you need to either:

```bash
# Option 1: Create the directory first with appropriate permissions
sudo mkdir -p /etc/blzbak
sudo chown $USER:$USER /etc/blzbak

# Then create sets normally
blzbak set create mybackup /path/to/backup

# Option 2: Run with sudo
sudo blzbak set create mybackup /path/to/backup
```

The cron job will be automatically installed with the schedule you specify (default is `0 2 * * *` - 2 AM daily).

To check your cron jobs:
```bash
crontab -l
```

You should see entries like:
```
0 2 * * * /path/to/blzbak backup run mybackup # blzbak-managed:mybackup
```
